# pom-insurance enabling-standards

Four starter standards installed by `pom-insurance-init` into the user's POM repo under `enabling/<concern>/`:

| Concern | Why it exists |
|---|---|
| `regulatory-reporting` | Tracks regulator-driven deadlines, state-filing matrix, and assigns ownership for keeping the watchlist current. |
| `actuarial-review` | Defines when actuarial sign-off is required, who provides it, and the artifact format (memorandum). |
| `policyholder-communications` | Bar for plain-language, accessibility, and consistency in policyholder-facing copy (claim, lapse, renewal, denial, rate-change letters). |
| `state-filings` | State-by-state filing scope mapping and SERFF tracking expectations. |

Each is a **starter**, not a finished standard. Each opens with scope, a baseline rule set, and open questions for the standard owner.

## Interaction with the generic baseline

`pom-core`'s generic enabling standards (ai-ethics, security-baseline, accessibility-baseline, data-quality) stay in place. These four additional concerns are insurance-specific extensions.
