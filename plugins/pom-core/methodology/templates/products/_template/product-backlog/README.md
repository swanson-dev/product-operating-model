# Product Backlog

Shaped opportunities ready for a Pod to pull. One markdown file per backlog item.

## Promotion in

An item arrives here only when its [`../discovery-backlog/`](../discovery-backlog/) entry has cleared all four questions: **desirable, viable, feasible, ethical/compliant**.

The Trio promotes — no one else.

## Promotion out (Pod pull)

Pods pull from the top of this backlog at their cadence. **PO Sync** (Trio + all pod POs) re-orders this backlog; pod POs do not silently reorder.

## File format

`PB-YYYY-MM-<short-slug>.md` — see [`PB-template.md`](PB-template.md). Each file contains:

- Link to originating Discovery Backlog entry
- **Outcome** the item delivers (measurable)
- **Vertical slice definition** — thin end-to-end customer journey, not a horizontal task list
- **Acceptance signals** — how the Trio will know the slice delivered the outcome
- **Runway prerequisites** — what must exist in [`../runway/`](../runway/) before pod pull
- Current state: shaped / pulled-by-`<pod>` / in-flight / shipped
